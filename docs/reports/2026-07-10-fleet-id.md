# Отчёт: Fleet-ID — «на чём поедешь» (B1–B5)

Дата: 2026-07-10 · Ветка: `feature/fleet-id` · Спека: `docs/FLEET_INTEGRATION.md`

Вся логика — на клиенте, данные — статический asset
`app/assets/data/fleet_models.json` (39 классов, 33 модели, 1089 машин
повагонно). Новых сетевых запросов нет. Backend, аналитика, SWR-кеш и
клиентский поллинг (≥30 s) не тронуты.

## Что сделано по пунктам

### B1. Матчер v2 — `app/lib/core/fleet_matcher.dart`
Чистый Dart без Flutter (по образцу `core/vehicle_track_animator.dart`),
юнит-тестится без обвязки.

- Алгоритм §2 буквально: `n = int(strip 'P')` → junk при `n < 1000` →
  точная повагонная карта `vehicles` (**model-hit**) → диапазоны классов
  (**class-hit**, при вложенности побеждает **узкий**, `min(b−a)`) → `UNKNOWN`.
- **Total**: любой вход даёт ровно один из `modelHit / classHit / unknown /
  unknownJunk`, никогда не бросает и не гадает «ближайший».
- Junk-правило совпадает с бэкендом (`backend/src/lib/analytics.ts:vehicleIdOf`,
  `/^P(\d+)$/` + `< 1000`).
- json парсится один раз (`FleetCatalog.tryParse`), лукап — по
  предвычисленному индексу (`Map` повагонно + список диапазонов), результат
  `resolve(n)` кешируется (номера иммутабельны в сессии).
- Различает model-hit / class-hit; несёт `approximate` для операторских
  классов частников (`confidence.attributes: per-vehicle`) и множество
  `assumedFields` для точечных `assumed` — это и есть «~» в UI (B2/B3).
- Model-hit обогащается `operator` и `type` из **узшего охватывающего**
  класса (в `models_catalog` этих полей нет; сама карта диапазон→оператор
  verified, это не догадка).

Тесты — `app/test/fleet_matcher_test.dart` (на реальном asset + синтетика):
P1–P999 → junk; граница 999/1000; повагонное > диапазона (`P70259` →
`conecto3_h`, оператор Strela); вложенность → узкий (`P23055` → `akia_banbus`,
не `banbus`); границы диапазона включительно (`P80201`/`P80399` → kt4); Lasta
(58xxx/76xxx/79xxx) → честный UNKNOWN; пустой/битый/без-`P` вход; вне всех
диапазонов → UNKNOWN; кеш возвращает тот же инстанс; проброс `assumed`.

### B2. Компакт-бейджи — `app/lib/presentation/widgets/fleet_badges.dart`
Бейджи подключены к **обоим** спискам прибытий: полноэкранный `StopScreen`
(deep-link `/stop/:id`) **и** шторка остановки на карте
`widgets/stop_sheet.dart` — это основной in-app путь при тапе по остановке.
(Первая итерация покрывала только `StopScreen`; шторка добавлена после проверки
на стенде. Регресс-тест — `test/fleet_stop_screen_test.dart` → «map stop sheet
also shows Fleet-ID badges».)

- В подписи строки прибытия, **одной строкой** рядом с «N stops away»:
  эко-иконка (электро — зелёная, гибрид — янтарная) + ❄️ (ac) + ♿ (low_floor)
  + возраст-бейдж (от середины `years_built`). Плотность списка не растёт.
- `per-vehicle`/`assumed` → приглушённо + ведущий «~».
- `UNKNOWN` → только приглушённый номер (`#…`), без модели и догадок.
- `UNKNOWN_JUNK` → строка-бейдж пустая, номер не показывается нигде.

### B3. Карточка модели — `app/lib/presentation/widgets/fleet_model_card.dart`
- Открывается тапом по бейджу (bottom-sheet, как `vehicle_detail_sheet`).
- Модель крупно (заголовок), прозвище мелко под ней (только ru/sr);
  `human_note_ru` — текст карточки.
- Атрибуты в порядке §3: ac → low_floor → возраст → comfort → articulated/
  capacity/length → powertrain (эко) → usb; в деталях — **«Производитель:
  CAF, Испания»** (`manufacturer`+`country`, только у конкретных моделей;
  названия стран локализованы EN/RU/SR — вкл. историч. CS/YU и составные DE/CH
  — таблицей в коде карточки, лейбл через `l10n.fleetManufacturer`) и
  `operator` («Перевозчик: …», только не-ГСП).
- `comfort_score` — не цифра: 5 точек + слово (`ретро/норм/комфорт`, §4).
- Assumed-поля — с «~»/серым; для `per-vehicle` внизу сноска-пояснение.
- Слот под визуал-героя (§8) заложен (плейсхолдер); карточка выходит без него.
- UI-строки локализованы EN/RU/SR (`lib/l10n/*.arb` + `flutter gen-l10n`).
- **Контентные поля json локализованы** (решение владельца, сверх исходной
  спеки §3, где было «как есть»): в схему добавлены `human_note_en/sr`
  (заметки классов), `note_en/sr` (`models_catalog`), `nickname_latin`
  (сербская латиница) и `nickname_en` (ASCII без диакритики). Матчер отдаёт
  `humanNoteFor(lang)` / `nicknameFor(lang)`: заметка — по локали с
  **fallback на `ru`**; **прозвище** в скрипте читателя — `ru` кириллица
  («Турчин»), `sr` латиница («Turčin»), `en` ASCII («Turcin», для колорита).
  В карточке **название модели — заголовок, прозвище мелко под ним**. Карточка берёт язык из `Localizations.localeOf`. Названия моделей
  (`name`) — имена собственные, не переводятся. Переводы (39 классов × EN/SR,
  9 латинских прозвищ, 11 model-заметок) написаны в рамках задачи: операторские
  классы — по шаблону, конкретные модели — вручную. Тесты —
  `fleet_matcher_test.dart` группа «localized content …» и
  `fleet_stop_screen_test.dart` (порядок в шапке, скрытие прозвища для `en`).

### B4. Сортировка «по комфорту» — `stop_screen.dart` + `widgets/stop_sheet.dart`
- **Выбранный контрол**: пара `ChoiceChip` «По времени» / «По комфорту»
  (в стиле существующих чипов фильтра линий на этом экране). Дефолт — время.
- Показывается **только** когда среди прибывающих ≥2 *разных* классов с
  известным `comfort_score` (иначе переключателя нет). Сортировка: комфорт ↓,
  тай-брейк по времени ↑.

### B5. Деградация — `fleetCatalogProvider` (`providers.dart`)
- Asset грузится один раз через `rootBundle`; `FleetCatalog.tryParse` → `null`
  при отсутствии/битом/невалидном по схеме json. `null` = вся фича молча
  выключена (нет бейджей, карточек, сортировки), транзит работает как раньше.
- Тесты: `tryParse` на битом/кривом по форме json → `null`
  (`fleet_matcher_test.dart`); UI-деградация при `null`-каталоге без падений
  (`fleet_stop_screen_test.dart`).

Плюс happy-path виджет-тесты (`fleet_stop_screen_test.dart`): бейджи ac/♿
рендерятся, junk-номер скрыт, чип комфорта появляется, сортировка поднимает
Bozankaya над Ката (кейс линии 12), тап по бейджу открывает карточку с
`nickname_sr` и заметкой.

## Как проверить

Предпосылка: `flutter test` и `flutter analyze lib test` зелёные (см. выше).
Фича живёт **в списке прибытий** (не на карте).

**В приложении (локально или на preview-стенде):**
1. Открыть любую остановку с активными прибытиями (тап по остановке на карте →
   шторка; либо deep-link `/stop/:id`, напр. Batutova `20091`, линия 79).
2. **B2 — бейджи:** рядом со временем компактная лента иконок — эко (электро/
   гибрид) · ❄️ (кондиционер) · ♿ (низкий пол) · возраст. У частников
   (per-vehicle) — приглушённо с «~». UNKNOWN → только номер; junk (P1–P999) →
   пусто.
3. **B3 — карточка:** тап по бейджу → шторка. Сверху **модель**, под ней мелко
   **прозвище** (ru кириллица / sr латиница / en ASCII). Атрибуты по §3,
   комфорт точками, строка «Производитель: CAF, Испания», перевозчик — у
   частников.
4. **B4 — сортировка:** на остановке с ≥2 машинами разных классов вверху
   появляются чипы «По времени / По комфорту»; переключение поднимает
   комфортную машину (кейс линии 12: Bozankaya над «Ката»).
5. **Локализация:** переключить язык (☰ → настройки) на English/Srpski —
   заметки, прозвища и страна меняются; где перевода нет — русский fallback.
6. **B5 — деградация:** если убрать/испортить `fleet_models.json`, фича молча
   выключается, транзит работает как прежде (покрыто тестом).

**Хорошие остановки:** частники-электро (Batutova, линия 79 — Ćurdić/Otokar);
центр (Trg Republike — ГСП-дизели); для контраста комфорта — трамвайные линии.

**Данные-инварианты** (`jq` по `app/assets/data/fleet_models.json`): 39 классов,
33 модели в `models_catalog`, 1089 машин в `vehicles`; все `vehicles`-ключи есть
в `models_catalog`; 3 вложенных электробусных диапазона (23050-23059 ⊂ banbus и
т.п.).

## Отклонения от спеки / решения
1. **«иконка типа» в бейдже (B2/§5)** свёрнута в уже существующий цветной
   аватар типа слева в строке — дублировать глиф в компакт-полосе смысла нет.
   Тип по-прежнему читается в строке; в полосе — только то, чего в ней ещё нет.
2. **Разные схемы у model-hit и class-hit.** `models_catalog` несёт
   `name/note_ru/years`, но не имеет `nickname_sr/usb/length_m/type/operator`;
   классы — наоборот. `FleetVehicle` объединяет обе в один тип со всеми
   nullable-полями; `type/operator` для model-hit берутся из охватывающего
   класса (см. B1). Расхождения промпта со спекой не было.
3. **Lasta 71000–71499.** Спека §6 числит блок «дырой», но в данных
   операторский блок `strela` 70000–71999 его покрывает — эти номера дают
   class-hit (Strela), а не UNKNOWN. Тест это фиксирует явно. Реальные дыры —
   58xxx и 76xxx/78xxx/79xxx (см. B6).
