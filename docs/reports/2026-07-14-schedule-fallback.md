# Отчёт: расписание-фолбэк, Фаза 1 (список плановых прибытий)

Ветка: `feature/gtfs-freshness` · Дата: 2026-07-14 · Статус: **Фаза 1 готова, на
staging. [STOP & ASK] перед Фазой 2 (карта scheduled-объектов). Контракт:
`docs/SCHEDULE_FALLBACK_CONTRACT.md`. Флаг `schedule_fallback` (OFF prod / ON
staging). В main/прод НЕ мержил.**

## Что делает
Заполняет дыру в ките «прибытия»: когда живого потока мало (ночь/межпик), экран
остановки больше не пустой — показывает плановые прибытия из GTFS-расписания,
как у Яндекса/Belgrade Bus. «Живой первый + расписание хвостом».

## Архитектура (precompute — воркер не парсит 110 МБ на запрос)
- **`build-schedule.mjs`** (в цепочке `npm run gtfs:build`): по каждой **городской**
  остановке — плановые отправления каждой линии/направления, сгруппированы по
  сервису RD/S/N. Город+пригород (600-е из пригородной версии, без задвоения).
  Формат: **минуты от полуночи** (overnight ≥1440 сохранены). Раздача per-stop
  (как shapes): `public/gtfs/schedule/<stop_id>.json`.
- **Размер (замерено, твой гейт):** 3239 файлов, 2.33 млн отправлений, **raw
  10.7 МБ / ~3.9 МБ gzip суммарно**; **per-request ~3.5 КБ raw / ~1 КБ gzip** —
  копейки на запрос (аудитория на мобильном). Минуты вместо секунд дали −26%.
- **`schedule/_meta.json`**: календарь (RD/S/N) + исключения `calendar_dates`
  (праздники), для резолва активного сервиса на дату.

## Логика (`src/lib/schedule.ts`, чистая/тестируемая)
- **Сервис-на-дату:** будни→RD, сб→S, вс→N + `calendar_dates` (праздник RD→N).
- **Таймзона:** «сейчас» считается в `Europe/Belgrade` (лето UTC+2 / зима +1).
- **Overnight:** времена ≥1440 (до 26:xx) от **вчерашнего** сервиса показываются
  в ранние часы сегодня; сегодняшний overnight (~24ч вперёд) отсекается потолком.
- **Окно:** следующие **3 рейса на линию ИЛИ 90 мин** (что больше), потолок 3ч,
  ≤10 на линию.
- **Дедуп live↔scheduled:** гасим плановый рейс, ближайший к каждому живому ТС на
  том же **направлении** (`direction_route_id` из фикса геометрии) в пределах
  20 мин; поздние плановые остаются хвостом. Не «есть живой → скрыли всё».

## Контракт
`ArrivalDto.source:"scheduled"` (аддитивно; live без изменений). Эмиссия за
флагом `schedule_fallback`. Экран остановки помечает плановые строки
(`arrivalScheduled` EN/RU/SR, приглушённо, иконка часов); пустого состояния нет,
когда по расписанию что-то идёт.
