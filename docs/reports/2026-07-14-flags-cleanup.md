## Сделано (безопасные пункты 1–3)

### 1. Живой реестр флагов — `docs/feature-flags.md`
Расширен из неполного (4 строки) в полный реестр всех **12 флагов** +
конфиг-параметр: `флаг | что гейтит | прод | staging | введён (дата) | статус`.
Статусы: **permanent** (за флагом по дизайну), **rollout** (раскатан и стабилен,
флаг стал рудиментом → кандидат), **fresh** (свежий, держим ради отката).
Добавлено **правило поддержания** (новый флаг → строка; удаляешь флаг → убери

### 2. Dead-code убран
- Удалён `scheduleFallbackEnabledProvider` (`providers.dart`) — нигде не
  watch-ался; клиент читает `.scheduleFallback` напрямую (`home_map_screen.dart`),

- **Проверены все прочие** флаг-провайдеры и геттеры: других мёртвых нет (каждый
  геттер/провайдер потребляется хотя бы одним экраном). Тесты: **flutter 176**,

«забыли завести»).
Заодно: удалены 3 мусорные `geocode:*` записи (клавиатурный мусор

### Почему «оставить пока» по всем
Сверка по дате (главный критерий из промпта): **все шесть раскатаны 13–14 июля —
им 0–1 день в проде.** Ни один не «давно ON». Окно отката флагом ценно именно
сейчас — первые дни реальной прод-нагрузки, когда всплывают баги. Удалять флаг
(впаивать путь намертво) через день после раската — снимать страховку в самый
нужный момент.

**Рекомендация:** пере-оценить примерно **2026-07-28** (≈2 недели стабильного ON
без инцидентов) — тогда `symbol_layer`/`timed_trajectory`/`live_position_only`/
`vehicle_direction_shape`/`schedule_fallback`/`schedule_map` станут чистыми
кандидатами: удалить флаг, впаять рабочий путь, убрать строку из реестра и
KV-ключи. Если по какому-то флагу ты уверен и хочешь убрать раньше — скажи по
нему отдельно, сделаю (удаление кода + чистка KV) отдельной задачей.
